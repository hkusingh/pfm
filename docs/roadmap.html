Personal Finance Manager

Phased Roadmap
Prepared for Harsh · June 4, 2026 · v0.1 · sequence-based (not calendar-committed)

Three phases take the product from a trustworthy shared budgeting tool to an AI-differentiated, multi-channel household finance platform. Phases are sequenced by dependency and value, not pinned to dates. AI features are woven into Phase 2 (the differentiator) and Phase 3 (depth and conversation).

Phasing principles

  • Habit before depth. Phase 1 ships the daily-use budgeting loop on a secure, shared foundation — the minimum that retains users.
  • Differentiate where it counts. Phase 2 spends the differentiation budget on AI insights, once there is enough enriched data to power them.
  • Broaden and converse. Phase 3 widens the 360° view, adds the conversational agent, and moves the habit to mobile.

2. Phase Details

Phase 1 MVP — Trusted shared view + budgeting

Goal: a household can connect accounts and run an accurate, shared budget — securely.

Core scope

  • Household, invites, configurable co-owner/member roles
  • Mandatory MFA (Google Authenticator + email), encryption, privacy controls
  • Account aggregation + enrichment; manual & CSV/OFX import
  • Per-account visibility (shared / private / balance-only); joint de-dup
  • Categories, sub-categories, monthly budgets, amortized sinking funds
  • Spending dashboard + default charts; recategorize & rules
  • Responsive web app

AI in this phase

Foundational, behind the scenes: smart categorization and recurring/annual detection (feeds sinking funds). No conversational surface yet — this is where the enriched data that powers Phase 2 gets created.

Dependencies

  • Aggregator selected (Plaid vs. MX benchmark)
  • Backend, data store, auth/MFA in place
Exit criteria: a two-person household can sign up with MFA, connect/import accounts with visibility controls, run a monthly budget with sub-categories and sinking funds, and see accurate shared spending — on the web.

Phase 2 Differentiate — AI insights + net worth

Goal: turn the data into proactive, plain-language guidance — the competitive edge.

Core scope

  • Household net worth + trend (assets & liabilities)
  • Property tags + rental cash-flow report (income − expenses = net, per property)
  • Custom chart builder; interactive report controls
  • Notifications (alerts, bills, insights); step-up auth for sensitive actions
  • Connection health, broader institution coverage, budget rollover

AI in this phase — the platform + first insights

  • AI platform: backend agent, read-only visibility-aware tools, model-abstraction layer (hosted + no-retention), guardrails, eval harness
  • Descriptive: anomaly alerts, plain-language monthly summary
  • Predictive: cash-flow forecast, bill prediction, sinking-fund shortfall alerts
  • Coaching: budget coaching from real behavior
Exit criteria: the household receives accurate proactive insights (anomalies, forecasts, monthly summary, budget coaching) and can see net worth — powered by the AI platform with guardrails and an eval harness in place.

Phase 3 Expand — Planning, investments, conversation, mobile

Goal: widen the 360° view, let users talk to their finances, and move the habit to mobile.

Core scope

  • Goals & scenario / what-if planning
  • Investment holdings, performance & allocation
  • Native mobile apps (iOS/Android)
  • Audit/access log; passkeys

AI in this phase — conversation + household intelligence

  • Conversational agent: ask-your-finances Q&A in plain English, returning charts — in-app chat first
  • Channels: Telegram / WhatsApp adapters (linked & redacted) as follow-on
  • Household intelligence: per-member patterns, fair-split & shared-goal insights (visibility-aware)
  • Coaching+: savings opportunities, opt-in spending nudges
  • Model evolution: begin hybrid / self-hosted migration using the eval harness
Exit criteria: users can converse with their finances in-app, get household-aware insights and planning, manage investments, and use the product on native mobile.

4. Key Assumptions & Risks

  • Aggregator choice (Plaid vs. MX) confirmed before Phase 1 build; enrichment quality affects AI accuracy.
  • AI data privacy rests on a no-retention provider agreement; revisit before scaling.
  • Cost management: route cheap tasks to small models; AI usage and aggregator fees scale with users.
  • Scope discipline: conversational AI and external channels are deliberately Phase 3 — resist pulling them earlier before the data and platform are solid.
