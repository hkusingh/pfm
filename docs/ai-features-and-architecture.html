AI is the product's chosen differentiator. This document captures (1) the AI feature set, grouped by capability and sequenced across releases, and (2) the architecture that enables them — data access, the agent, model strategy, and how users interact in natural language.

2. Cross-cutting Guardrails

These apply to every AI feature and are part of the architecture, not an afterthought.

  • Informational, not advice. Output is framed as information and suggestions; the product is not a financial or legal advisor and says so.
  • Read-only by design. The AI can analyze and suggest but can never move money, place trades, or initiate transfers — those remain manual user actions.
  • Visibility-aware. The AI only ever sees data the asking user is permitted to see, enforced at the tool layer (per the household sharing rules).
  • Transparent & explainable. Insights show why they appeared and what data drove them.
  • Correctable. Users can dismiss or correct insights; the system learns and avoids being noisy.
  • Privacy-first data handling. No-retention/no-training agreements with providers; minimize what data is sent; full audit of AI data access.

3. Architecture Overview

4. How AI Accesses the Data

The model never receives raw transaction dumps — that is costly, leaks data, and doesn’t scale. Instead it uses tool calling: the model is given structured tools and calls only the ones it needs.

  • Structured tools like get_transactions(filters), get_budget_status(), forecast_cashflow(), get_net_worth().
  • Shared query layer. These tools are the same metric/dimension engine that powers the custom chart builder — one query layer serves both charts and AI.
  • Retrieval (RAG) adds context (the household’s categories, history) where descriptive reasoning needs it.
  • Every tool is read-only and visibility-aware — the guardrails are enforced here, at the data boundary.

5. The Agent

An agent orchestrates multi-step reasoning. Simple questions are a single tool call; complex ones run a reason → call tool → observe → continue loop. The same agent serves both user-initiated chat and scheduler-initiated proactive insights — one brain, two triggers.

6. Model Strategy

A model abstraction layer sits between the agent and any LLM, so the provider is a swappable component. Ship with one model behind it; add routing and providers later without touching tools, agent, prompts, or guardrails.

Privacy posture & the hosted → self-hosted path

StageApproachTrade-off
LaunchHosted frontier API under a zero-retention / no-training agreementBest quality, fastest; relies on a vendor agreement
TransitionHybrid — move cheap, high-volume tasks (categorization, recurring) to self-hosted open models; keep complex reasoning hostedProves out ops gradually; lower risk
LaterSelf-hosted open models for more/all workloadsMax privacy/control; GPU hosting, scaling & monitoring burden
What makes the migration safe: build an evaluation harness now (representative queries + expected outcomes) to compare models objectively before switching; depend only on capabilities open models also support (notably reliable tool/function calling); keep a clean data/tool boundary. Then the move is “write a new adapter and re-point config,” not a rewrite.
The real cost of self-hosting is operational, not architectural — GPU hosting and its cost, scaling/latency, and monitoring. The abstraction layer lets you defer that burden without penalty.

7. Natural-Language Channels

Because the AI is a backend service with a clean API, any channel can talk to it. Launch: in-app chat in the mobile and web apps — the most secure, full-fidelity surface we control. Follow-on: external messengers via thin channel adapters.

What follow-on channels require

  • Secure account linking. A messenger user must be verified and bound to their PFM account and permissions (one-time linking).
  • Privacy on third-party platforms. Telegram bot messages are not end-to-end encrypted, so detailed balances over external chat is a real consideration. Approach: redacted/summary replies on external channels, opt-in required, and treat the in-app chat as the full-fidelity surface.
