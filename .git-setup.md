## First-time setup (from existing local files)

```bash
mkdir -p ~/projects/pfm
cp -R "/Users/harsh/Documents/Claude/Projects/Personal Finance Manager/." ~/projects/pfm/
cd ~/projects/pfm
git init
git add .
git commit -m "Add PFM planning artifacts and Phase 1 epics"
git branch -M main
git remote add origin git@github.com:hkusingh/pfm.git
git push -u origin main
```

> If GitHub rejects the push because the remote already has commits (e.g. you created the repo with a README/license), overwrite with your local content:
> ```bash
> git push -u origin main --force
> ```
> To avoid this, create the GitHub repo **empty** (no README/.gitignore/license).

## Scaffold the epic branches (parallel work)

One long-lived branch per Phase 1 epic. Run after the first `main` push.

```bash
cd ~/projects/pfm
for b in 0-foundation 1-household 2-accounts 3-import 4-categories 5-transactions 6-budgets 7-dashboard; do
  git branch "epic/$b" main
  git push -u origin "epic/$b"
done
```

## Branch conventions

- `main` — protected; always green; releasable.
- `epic/<n>-<name>` — long-lived, one per epic/team (e.g. `epic/2-accounts`).
- `story/<id>-<slug>` — short-lived; PR into the epic branch (e.g. `story/E2.1-plaid-link`).

Typical developer flow:

```bash
git checkout epic/2-accounts
git checkout -b story/E2.1-plaid-link
# build, commit
# open PR: story/E2.1-plaid-link -> epic/2-accounts
```

Epic branches merge into `main` at the wave boundaries (see `CONTRIBUTING.md`).
